# ADR-002: Trailing-Slash Canonical URL Policy

## Status

Accepted

## Context

GitHub Pages serves directory `index.html` files and issues HTTP 301 redirects for any
URL path that lacks a trailing slash (e.g., `/texas/keller` → `/texas/keller/`). These
redirects inflate crawl budget, split PageRank, and create inconsistency between the
React SPA router and the static pre-render layer.

## Decision

All non-root published URLs MUST use trailing-slash canonical form (e.g., `/texas/keller/`).
A `withTrailingSlash()` utility enforces this at every link-generation site:
`seo-utils.ts`, `generate-static-pages.js`, and `generate-sitemap.js`. The React router
strips trailing slashes only for internal Wouter route matching; all `href` attributes and
canonical tags always include the trailing slash.

## Consequences

### Positive

- Zero 301 redirects from internal links; crawlers follow direct 200 responses.
- Canonical tags, sitemap entries, og:url, and structured-data URLs are consistent.
- Complies with GitHub Pages directory-index serving model.

### Negative

- Requires discipline: every new link-generation site must call `withTrailingSlash()`.
  A missed call in the home-page section articles caused a regression (fixed in commit `478b25d`).
