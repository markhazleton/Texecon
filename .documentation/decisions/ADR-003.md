# ADR-003: Tailwind CSS v4 CSS-First Configuration

## Status

Accepted

## Context

Tailwind CSS v4 changed its configuration model from a JavaScript config file to a
CSS-first approach, removing the need for `postcss.config.js` and `tailwind.config.js`.
The project migrated from v3 to v4 to benefit from improved build performance and
smaller bundle output via the `@tailwindcss/vite` Vite plugin.

## Decision

Tailwind CSS v4 is integrated via `@tailwindcss/vite` as a Vite plugin. Configuration
is expressed directly in CSS rather than in a JavaScript config file. The PostCSS
dependency was removed. The previous `tailwind.config.ts` is retained as a backup
(`*.v3-backup`) for reference only.

## Consequences

### Positive

- Faster builds — Tailwind v4's Oxide engine is significantly faster than v3 JIT.
- Simpler dependency tree — no PostCSS config needed.
- CSS-first configuration aligns with standard CSS authoring practices.

### Negative

- Breaking change for team members unfamiliar with v4 syntax — contrast and spacing
  utilities changed; required manual contrast fixes post-migration.
- Some v3 utility class names changed; migration required a search-and-replace pass.
