# ADR-001: Static-Site Generation via Vite + GitHub Pages

## Status

Accepted

## Context

Texecon needed a hosting model that required zero ongoing server costs, was trivially scalable,
and matched the read-heavy nature of economic analysis content. A server-backed runtime would
add operational overhead without providing value for this workload.

## Decision

The production artifact is fully static HTML, CSS, and JavaScript published to GitHub Pages.
Vite 8 handles bundling and build-time asset emission. A custom `generate-static-pages.js`
post-build script creates crawlable `index.html` files for every route derived from the
WebSpark content API.

## Consequences

### Positive

- Zero hosting cost (GitHub Pages free tier).
- Content is served at CDN-edge latency.
- No server attack surface; security posture is purely static file delivery.
- Build artifacts in `target/` are fully reproducible and auditable.

### Negative

- Dynamic server-side features (e.g., real-time data, personalization) are not possible without
  a separate API layer.
- Every content update requires a full build + deploy cycle.
