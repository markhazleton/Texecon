## Highlights

Texecon v1.0.0 is the first sealed release of the TexEcon Texas Economic Analysis
site. The repository ships a fully static website published to GitHub Pages at
texecon.com. Content is fetched at build time from the WebSpark API and transformed
into crawlable HTML pages, a sitemap, and a structured navigation hierarchy.

This release includes the complete site architecture: React 19 SPA with server-side
pre-rendered static HTML for crawlers, Tailwind CSS v4, Wouter routing, Vitest test
coverage, Lighthouse CI integration, and a spec-driven DevSpark v2.4.0 workflow.

A key SEO fix landed in this release: all internal links now consistently emit
trailing slashes (`/texas/keller/`) matching the GitHub Pages canonical URL shape,
eliminating a class of 301 redirects that was degrading crawl efficiency.

## New Features

### Static Site Generation Pipeline

The full build pipeline (`clean → fetch:content → generate:sitemap → vite build →
generate:static-pages`) produces a deployable artifact set in `target/`. Each content
route from the WebSpark API gets a pre-rendered `index.html` with real anchor links,
breadcrumbs, and structured data for search engine crawlers.

### Dynamic Navigation and SEO

Navigation menus, breadcrumbs, footer links, canonical URLs, Open Graph tags, Twitter
Card tags, and `sitemap.xml` all derive from the same WebSpark content payload,
ensuring consistency across all SEO surfaces.

### Tailwind CSS v4

Migrated from Tailwind v3 to v4 using the `@tailwindcss/vite` plugin and CSS-first
configuration, removing the PostCSS dependency and improving build performance.

### DevSpark Spec-Driven Workflow

DevSpark v2.4.0 is integrated with full slash-command support in Claude Code, including
the new `commit-audit`, `address-pr-review`, and `update-pr` commands and the Agent
Skills surface (`write-spec` skill).

### GitHub Pages CI/CD

Automated build-and-deploy pipeline via GitHub Actions with Lighthouse CI integration,
security audits, type checking, and environment-aware base-path handling for both
custom-domain and project-pages deploy modes.

## Bug Fixes

- **Trailing-slash 301s**: Home page article section links (`/texas`, `/texas/keller`,
  etc.) were generated without trailing slashes, causing GitHub Pages to issue 301
  redirects for every crawler and user click. Fixed in `generate-static-pages.js` by
  wrapping all `escapeHtmlAttr()` calls with `withTrailingSlash()`.
