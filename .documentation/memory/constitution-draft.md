## Core Principles

### I. Simplicity (NON-NEGOTIABLE)

Prefer simple, readable solutions over clever or complex ones.
Abstractions must be justified by concrete reuse — not hypothetical future needs.
When in doubt, do less and do it clearly.

*Discovered from*: Lightweight routing (Wouter over React Router), no server runtime, static-first architecture.

---

### II. Explicit Over Implicit (NON-NEGOTIABLE)

Behavior must be predictable and visible.
Avoid magic, hidden side effects, and implicit conventions that require tribal knowledge.
Configuration over convention where clarity is at stake.

*Discovered from*: `exactOptionalPropertyTypes: true` in tsconfig, explicit `@/*` path aliases, API-driven URL/navigation conventions documented in AGENTS.md.

### IX. Static-First Architecture (NON-NEGOTIABLE)

- There is no server runtime — all output is static HTML/JS/CSS in `target/`
- Build output directory is `target/` (not `dist/` or `build/`) — configurable via `TARGET_DIR`
- The full build pipeline **MUST** be preserved: `clean → fetch:content → generate:sitemap → vite build → generate:static-pages`
- Client-side routing fallback via `404.html` **MUST** remain functional
- `target/` artifacts **MUST NOT** be manually edited — regenerate via build scripts

*Evidence*: `package.json` build script, AGENTS.md architecture documentation.

### X. SEO and Crawlability (MANDATORY)

- All pages **MUST** have complete meta tags (title, description, canonical, OG, Twitter)

- Static HTML files **MUST** be generated for all dynamic routes
